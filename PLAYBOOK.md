# Плейбук: как устроен сайт и как добавить новый квартирник

Этот файл — инструкция, чтобы **не объяснять всё заново в новом чате**. Дай ссылку на него (или на весь репозиторий) любому ИИ-агенту или человеку — и он поймёт систему за минуту.

## Главная идея

Весь «переменный» контент сайта живёт в **одном файле** — `data/events.json`. Страницы `events.html`, `event.html` и `archive.html` собираются из него автоматически на JavaScript. HTML руками править не нужно — только JSON.

Жизненный цикл одного квартирника:

```
[анонс] → events.html показывает его как "Следующее мероприятие"
   │        (статус "upcoming", регистрация)
   ▼
[событие прошло] → меняем статус на "past", добавляем фото, песни,
   │                ссылку на папку Google Диск
   ▼
[архив] → та же запись автоматически уезжает в archive.html,
           открывается как страница прошедшего события
```

Одна и та же страница `event.html?id=…` обслуживает и будущее, и прошедшее событие — это и есть «страница следующего события превращается в страницу прошедшего».

## Как добавить НОВЫЙ квартирник (5 минут)

1. Открой `data/events.json`.
2. Добавь **новый блок в начало** массива `events`:

```json
{
  "slug": "2026-05-30",
  "date": "2026-05-30",
  "date_label": "30 мая 2026",
  "title": "",
  "place": "Москва, БЦ Лотте",
  "place_note": "адрес появится ближе к дате",
  "status": "upcoming",
  "entry": "Бесплатно · донат приветствуется",
  "cover": "https://.../обложка.jpg",
  "photos": [],
  "songs": [],
  "gdrive": "",
  "register_url": ""
}
```

3. Если предыдущее событие было `"upcoming"` — поменяй ему `status` на `"past"`.
4. Сохрани, закоммить, запушь. GitHub Pages опубликует сам.

> `slug` — латиницей, уникальный (обычно дата). `date` — строго `ГГГГ-ММ-ДД` (по ней сайт понимает, прошло событие или нет). `date_label` — как показать человеку.

## Как «закрыть» прошедший квартирник

У той же записи:
- `"status": "past"`
- `photos`: 3–6 лучших фото (ссылки на изображения);
- `songs`: список песен, которые играли (массив строк);
- `gdrive`: ссылка на папку события в Google Диске (все фото/видео/аудио);
- `register_url` можно убрать.

## Google Диск: структура папок

Родитель: **«Квартирник у Маши»** (общий с этого Gmail).
Внутри — папка события, названная по дате. Рекомендуемая структура каждой папки события:

```
Квартирник у Маши/
└── 2026-05-30 БЦ Лотте/
    ├── Фото/
    ├── Видео/
    └── Аудио/
```

Папку события расшарить «всем, у кого есть ссылка → просмотр», ссылку вписать в поле `gdrive`.

## Разбор Telegram-чата (источник дат, песен, медиа)

Экспорт чата лежит в Google Диске: папка `ChatExport_2026-04-25`, файлы `messages.html … messages7.html` (переписка 15.07.2022 – 25.05.2025).

Что из него берётся:
- **даты событий** — по служебным сообщениям и анонсам «Квартирник … числа»;
- **песни** — из сообщений и ссылок на таблицы с программой;
- **фото/видео** — ⚠️ в текущем экспорте медиа помечены «Not included». Чтобы получить файлы, нужен **повторный экспорт из Telegram Desktop с галочками Photos / Video / Voice messages** (Settings → Advanced → Export Telegram data → отметить медиа). Затем медиа раскладываются по папкам событий на Диске.
