# Бэклог: простая CMS для сайта квартирника

Цель: **редактировать контент сайта легко, руками, без кода** — добавить текст, ссылку, фото к событию. Доступ сначала только с этого Gmail (Flatnik). Бесплатно или очень дёшево. Полный контроль — данные и код остаются у тебя.

Ниже — варианты (от самого простого) и задачи. Это для обсуждения: сначала выбираем подход, потом делаю.

---

## Три варианта CMS (выбрать один)

### Вариант A. Правка `events.json` через Google-таблицу → авто-сборка (рекомендую для старта)
Ты ведёшь события в **Google-таблице** (привычно, доступ по Gmail, правится с телефона). Скрипт превращает таблицу в `events.json` и пушит в GitHub → сайт обновляется сам.
- 💚 Плюсы: ноль нового кода на сайте, редактируешь как обычную таблицу, бесплатно.
- 🟡 Минусы: нужна разовая настройка связки «таблица → GitHub» (GitHub Action по расписанию или кнопке).
- 💰 0 ₽.

### Вариант B. Decap CMS (бывш. Netlify CMS) — админка прямо на сайте
Отдельная страница `/admin` с формами: заголовок, дата, фото, песни, ссылка на Диск. Сохраняет прямо в `events.json` в репозитории через GitHub.
- 💚 Плюсы: настоящая «админка», удобные формы, история изменений в Git, доступ ограничивается твоим GitHub/Gmail.
- 🟡 Минусы: разовая настройка авторизации (GitHub OAuth), чуть больше «магии».
- 💰 0 ₽ (open-source, хостинг на GitHub Pages).

### Вариант C. Notion как источник → сайт (ты уже живёшь в Notion)
Контент событий ведёшь в базе Notion, скрипт тянет её в `events.json`.
- 💚 Плюсы: редактируешь в знакомом Notion, фото прямо туда.
- 🟡 Минусы: нужен Notion API-ключ и периодическая синхронизация; фото из Notion недолговечны по ссылкам — придётся переносить.
- 💰 0 ₽ на текущем объёме.

**Рекомендация:** начать с **A** (быстро, привычно, ничего не ломает), а если захочется «настоящую админку» — перейти на **B**. Обсудим на созвоне.
